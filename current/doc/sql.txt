Quindi il parser SQL è in realtà un compilatore di espressioni che generano SQL.

Questo chiarisce anche la distinzione:

un campo produce testo SQL direttamente;
una costante produce testo SQL già quotato;
una variabile esterna produce codice VM che legge il valore e lo quota;
una funzione EWB indipendente dal record produce codice VM che la esegue e quota il risultato;
una funzione applicata a un campo deve avere una traduzione SQL oppure è errore.
