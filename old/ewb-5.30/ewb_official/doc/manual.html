Concetti base del linguaggio Basical conception

Un programma easyweb e' un file di testo con estensione .ewb , che va compilato con il compilatore "ewb". The easyweb rogram is a test file with extension ".ewb", that will be compiled with the "ewb" compiler.
Un programma e' composto da definizione di funzioni e codice. The program is made by functions and the main code.
Normalmente, le funzioni che verranno utilizzate in una parte del codice sono definite prima che il codice inizi, ma e' possibile inserire liberamente linee di codice e definizione di funzioni, senza un ordine preciso. Normally the functions are inserted before the main code, but if you want, you can freely insert main code lines or new functions without an order.
Una funzione viene definita con la parola chiave sub: A new function starts with the keyword "sub"

sub nomefunzionename (parametro, parametroparameter, parameter .... )
{ codice della funzionecode of the function
};

Il codice e' composto di istruzioni e puo' contenere commenti. The code is made by instructions and comments.
Un commento e' una istruzione che inizia con un apice, e su questa line si puo' scrivere qualunque cosa, fino al caporiga. Il contenuto della linea viene  ignorato dal compilatore. A comment is a line that starts with " ' ". On that line you can write everything, and the compiler will ignore it, till the end of line char.
Una istruzione e' sempre terminata da un punto e virgola. Ad esempio print "ciao\n"; e' una istruzione. Un programma minimale e' composto di almeno una istruzione. Each instruction ends with a " ; ". For example print "ciao\n"; is an instruction. A minimal program is composed by at least one instruction.
Se una istruzione utilizza delle variabili, queste devono per prima cosa essere dichiarate. If an instruction use some variabile, you have to declare that variabile on a previous instruction.
Esistono due tipi di dichiarazione di variabili: inplicita ed esplicita. There are two ways to declare a variable: implicit or explicit.
Il caso di dichiarazione esplicita e' quello in cui viene indicata la parola chiave var seguita dall' elenco delle variabili da dichiarare: On the esplicit way, you write the keyword "var" followed by the variabile name(s):
var alfa, beta, gamma;
Dichiara le tre variabiliThis declare three variabiles: alfa, beta e gamma.
Esistono due casi di dichiarazione implicita: There are two implicit-declaration cases:
1 - dichiarazione all'apertura di una funzionethe opening of a function:
sub prova(uno, due)
{
'dentro questa funzione sono definite le variabili "uno" e "due", ma non esistono piu' dopo la chiusa graffa di fine funzione. In that function we have two variables: "uno" and "due", but they disappear after che end of the function.
};
2 - dichiarazione alla dichiarazione di una tabella. The declaration of a table.
table presone=nome, cognome, indirizzo;
Dopo questa riga sono dichiarate le variabili After that row there are new variabiles: "nome", "cognome", "indirizzo".
Una seconda dichiarazione di una variabile gia' dichiarata non causa un errore, ma semplicemente azzera il valore della variabile. Repeating a variabile declaration you won't receive an error, but simply the value of the variabile will be zero.
Una dichiarazione all' interno di un blocco di una variabile che gia' esista esternamente non ne cambia il valore, ma maschera solamente la variabile esterna, dimodoche' nel blocco non la si puo' utilizzare.

var a;
a=4;
sub prova()
{ var a;
   a=5;
};
print a;  'qui a vale ancora 4.
 
Il linguaggio easyweb contiene un preprocessore simile a quello del linguaggio C. Il preprocessore puo' essere utilizzato per spezzare il codice in piu' files, creare delle librerie di funzioni, o avere del codice che venga compilato solo se sussistono certe condizioni e sostituito con altro codice se le condizioni non sussistono.
Lo stesso preprocessore puo' essere utilizzato per definire costanti, da utilizzare poi nel codice.
La sintassi del preprocessore e'  la stessa del preprocessore del linguaggio C.

La compilazione e' seguita da una fase di link che rende il piu' possibile compatto il file risultante, mentre nelle versioni di easyweb precedenti alla 3.0, praticamente ogni programma occupava piu' di 40 kilobytes solo per lo spazio occupato dalle librerie. 

Easyweb e' un linguaggio che puo' essere utilizzato sia per realizzare applicazioni da console, come ad esempio il C, sia per realizzare applicazioni per piattaforma web.

Il linguaggio Easyweb e' pensato per essere concettualmente differente da linguaggi come perl, php, asp, che sono solo un ausilio per rendere dinamico il web.
In un linguaggio per pagine dinamiche, come asp, un file asp non descrive una applicazione, ma semplicemente una pagina dell' applicazione.
L' applicazione viene costruita come un insieme di pagine distinte, di cui alcune dinamiche e altre no, e le informazioni devono essere passate continuamente da una pagina all' altra mentre l' utente naviga.
Inoltre l' interfaccia grafica dell' applicazione e' solo il web: allora per poter fare all' utente una richiesta e' necessario costruire la pagina web che ne contenga i campi, e la corrispondente pagina asp che si occupi di controllarli ed utilizzarli.
Questo significa che ci sono problemi di concorrenza tra il contenuto dei form html e il contenuto delle pagine asp che rispondono a questi form, e c'e' disordine nell' applicazione: una applicazione e' contenuta solitamente in una cartella, ma all' interno di questa cartella e' composta di molti piccoli files distinti.
Per fare un esempio, una semplice applicazione che raccolga un nome e un numero di telefono e li registri in una tabella deve essere composta di tre files:
- un file di inserimento del numero di telefono.
- un file che legge i valori del numero inserito, li controlla e fornisce la risposta
- l' html di risposta
- oltre a questi abbiamo solitamente un file di database che contega la tabella, collegato in qualche modo all' applicazione.

Un altro problema dello sviluppo dell' applicazione in questo modo e' che non esistono costrutti di linguaggio che permettano di accedere in maniera semplice ad un database:
Per accedere ad un database e' solitamente necessario utilizzare una libreria di interfaccia esplicita ad odbc, scrivere esplicitamente le richieste sql e gestire come liste le risposte, sempre che il linguaggio dia la possibilita' di gestire liste.
Per utilizzare una tabella di database dalla mia applicazione e' necessario che per prima cosa l' amministratore del database abbia creato l' intera struttura di database che io intendo utilizzare e me ne abbia dato i diritti.
Questo meccanismo, per quanto molto ordinato e' comunque estremamente complesso da applicare, e richiede un intervento esterno perche' l' applicazione possa girare.

EasyWEB risolve i due problemi a questo modo:

Per quello che riguarda la necessita' di spezzare una singola applicazione in piu' files, easyweb permette di descrivere l' interazione con il web server con un unico costrutto, analogo al comando input di un interprete basic.
L' utente dell' applicazione scrivera' qualcosa di analogo alle seguenti righe:

utente="nobody";
pass="";
ask utente,pass;
if ((utente="a") && (pass=b)) ...

In realta' quello che succede e' che easyweb lancia una prima volta l' applicazione, crea un form con cui inviare la richiesta all' utente e si ferma.
Una volta che l' utente abbia risposto a questo form l' applicazione viene ripresa da dopo la ask, con le variabili valorizzate, e l' elaborazione continua come se il programma fosse unico.
Quello che abbiamo ottenuto e' intrinsecamente un'applicazione multi-tasking il cui singolo flusso di esecuzione e' anche esso spezzato in piu' processi.
 

Per quello che riguarda la necessita' di accedere in maniera semplice alle tabelle di un data base relazionale, la soluzione e' un po' discutibile: viene infatti implementato il piu' ridotto possibile insieme di istruzioni di accesso al database per poi lasciare al software la combinazione di queste:

Normalmente in un database si memorizzano tabelle, ciascuna composta di piu' colonne,  e ogni riga di tabella contiene la nostra unità di informazione: ad esempio

nome
 cognome
 telefono
Enrico

L' intera tabella con i nomi dei campi viene detta relazione, e la singola riga, il cui significato e' ad esempio nome=enrico, cognome=betti, numero=113 viene detta t-upla , in quanto composta da t elementi.

Una relazione e' semplicemente un insieme di tuple. Questo significa che solitamente le righe della tabella non hanno un particolare ordine: se avessi scambiato la t-upla "enrico, betti, 113" con la t-upla "andrea,rossi,418965" , avrei comunque le stesse informazioni.

Il collegamento tra una tupla e il nome della tupla stessa non fa parte della relazione: e' una informazione riguardante il programma applicativo. Questo almeno in easyweb.
Nel mio programma devo quindi essere io ad indicare un nome per le informazioni contenute in una tabella.
table utenti= nome, cognome, password;
Da qui in poi nel programma esistono tre variabili: nome cognome e password, che possono essere utilizzate ad esempio per calcoli:
Non e' possibile usare variabili se prima queste non sono state dichiarate, e l' uso del costrutto table equivale ad una dichiarazione.

Dato che le variabili fanno parte di una tabella, possono essere valorizzate con quella che noi vogliamo sia una t-upla valida nella tabella utenti:

A questo punto e' possibile inserire la t-upla nella tabella semplicemente con l' istruzione

add(utenti);

Allo stesso modo, quando andremo a leggere un dato dalla tabella, quello che succederà e' che cambieranno i valori di nome, cognome e telefono per ritornare il dato richiesto.

Si e' scelto di non tipizzare le variabili, e quindi i dati contenuti nelle tabelle.
Questo significa che una variabile una volta dichiarata puo' essere utilizzata per contenere un dato di qualunque tipo, quindi ad esempio una stringa, un numero, un carattere, un valore booleano vero/falso, un testo di piu' righe o un file, o anche una lista di variabili.
La stessa cosa vale per una riga della tabella.
Vedremo piu' avanti come funziona questo meccanismo.

Abbiamo detto che una applicazione easyweb non e' un programma singolo, ma ne possono girare piu' istanze contemporaneamente, e ciascuna di queste puo' essere ad un punto diverso della comunicazione con il proprio utente.
Se l' accesso alla tabella degli utenti avvenisse contemporaneamente per due copie dell' applicazione, potremmo leggere dati inconsistenti, e questo potrebbe essere causa di problemi, incomprensioni o perdita di dati.
Ad esempio potrebbe accadere che una altra istanza della mia applicazione stia leggendo le righe della tabella mentre la mia istanza sta inserendo su questa un nuovo dato, e questo potrebbe portare alla lettura di dati incompleti o errati.
Per questo easyweb prima di accedere ad una tabella la blocca, cioe' ne richiede l' uso esclusivo per una istanza dell' applicazione, o se la tabella e' già in uso attende che questa sia libera prima di utilizzarla.
Lo stesso problema si puo' pero' porre per un aggiornamento che coinvolga piu' tabelle: ad esempio se dovessi spostare del denaro da un conto in banca ad un altro dovrei probabilmente svolgere due operazioni: la prima e' togliere la cifra dalla tabella dei miei conti, e la seconda inserire la stessa cifra nella tabella di conto di destinazione.
Anche se le due tabelle vengono bloccate una alla volta, esiste una fase intermedia in cui sono entrambe sbloccate, e il denaro non si trova ne' in un conto ne' nell' altro. Cosa accadrebbe se in questo momento la tabella venisse usata da un programma di revisione di conti? sicuramente segnalerebbe un ammanco di cassa.
Deve allora essere possibile bloccare piu' tabelle contemporaneamente, e ancora sbloccarle contemporaneamente, per svolgere quella che viene chiamata una "transazione".
Questo viene fatto utilizzando le istruzioni lock e unlock.
Riporto come esempio il codice che cancella una persona dalla tabella "conoscenti" e la sposta nella tabella "amici" per il telefono.

table conoscenti=nomecognome, telefono;
table amici=nomecognome, telefono;
'notate che ho usato lo stesso nome per le voci di entrambe le tabelle.

nomecognome="Andrea Rossi";
lock(conoscenti);
lock(amici);

delete(conoscenti);
add(amici);

unlock(amici);
unlock(conoscenti);

Dopo una lock la tabella conoscenti e amici possono essere utilizzate solo dall' istanza corrente del programma.
E' necessario ricordarsi di sbloccare le tabelle, altrimenti altri per poterle usare dovranno attendere. Un meccanismo di sicurezza fa comunque in modo che il lock venga rimosso dopo un certo tempo, impedendo ad un programma errato di bloccare tutto il sistema.

Un lock non puo' rimanere attivo durante una ask, perche' questo significherebbe bloccare la tabella per un tempo indefinito.
Il lock serve quindi a gestire azioni transazionali su piu' tabelle contemporanee.

Sintassi

La sintassi di easyweb viene riportata come insieme di produzioni nel file sintassi.txt. Questo file e' stato il mio riferimento durante tutta la fase di sviluppo del linguaggio stesso ed e' a tutt' oggi valido.
In una variabile puo' essere inserito un qualunque valore, indipendentemente dalle sue dimensioni. Queste, che nella maggior parte dei casi e' una semplificazione, (basti pensare alle tabelle di database) diventa una complicazione nel momento in cui si usano operatori per svolgere i calcoli. Vedremo operatori diversi per ogni tipo di dati.

Operatore di assegnamento =
Viene utilizzato per assegnare ad una variabile un valore. Da notare che non e' lo stesso operatore utilizzato per confrontare una variabile con un valore come si usa in basic.
a="3";
if (a=1) {
    'questo codice verrà sempre eseguito, e il valore di a diventerà 1: cosi' il confronto non funziona.
 };

Operatori di confronto: ==, >, < , > >=, <=, eq, ne, ge, gt, le, lt

== (uguale), < (minore), <(maggiore), >=(maggiore o uguale), <= (minore o uguale): tutti questi sono operatori da utilizzare per confrontare tra loro numeri interi o valori booleani. ad esempio:
a="3";
b=5;
if (a<b) {
'questo confronto e' vero, perche 3<5  
};
if (a==b) {
'questo confronto e' falso, perche' 3 non e' uguale a 5
};
Sono da notare alcune cose:
Per prima cosa abbiamo scritto i numeri sia con che senza le virgolette. Questo perche' nella variabile i numeri vengono memorizzati come stringhe, cioe' come sequenze di caratteri.
Questo vuol dire anche che una stringa verra' accettata come un numero, prendendone tutte le cifre valide.
Ad esempio "3prova" corrisponde al numero 3, mentre "parola" corrisponde al numero 0.
Allo stesso modo un valore booleano "vero" e' rappresentato da una qualunque stringa non vuota, mentre il falso e' la stringa vuota "".

Operatore concatena   . 

L' operatore concatena serve a concatenare piu' stringhe, cioe' a mettere i caratteri della seconda in coda ai caratteri della prima. 

Ad esempio:  

a="3.1421";
print "prova". [ di scrittura del numero]. a."\n";

Scriverà a video "prova di scrittura del numero 3.1421 e poi andra' a capo.

Operatori algebrici: +, -, *, / ,%

Sono i normali operatori, che possiamo trovare su una qualunque calcolatrice, cioe' piu', meno, per e diviso. L' ultimo e' l' operatore modulo, che torna il resto di una divisione intera.
Cosi'
print 2+2; 'scrive 4
print 2-2;  'scrive 0
print 3*2; 'scrive 3
print  9/2; scrive 4.5
print 10/6; scrive il resto di 20/6, cioe' 2.

Operatori di or e and ||, |, &&, & 

Sono gli operatori utilizzati per un confronto tra valori booleani, cioe' ad esempio all' interno di una condizione. 
|| e' l' or logico: ad esempio la condizione "se a e' minore di dieci oppure a e' maggiore di venti" si scrive cosi':

if ( (a<10) || (a>20)) { codice };

&& e' l' and logico, quindi
if ((a<20) && (a>10)) {};  significa "se a e' compreso tra dieci e venti, estremi esclusi.

| e' l' or bit a bit sui valori della variabile, letta in binario.
Ad esempio 3 | 6 = 7  (0011b  | 0110b = 0111b)
& e' l' and bit a bit fatto allo stesso modo.
 3 & 6 = 2  (0011b & 0110b = 0010b)

Operatore negazione ! 

L' operatore negazione inverte il valore di una variabile booleana. ad esempio:
if (! (a<3)) {};
la condizione e' "se non (a minore di tre), quindi se a e' maggiore o uguale a tre.

L'operatore di confronto ha la precedenza sugli operatori di confronto. Gli operatori di confronto hanno la precedenza sugli operatori algebrici, e questi sugli operatori di or, and e sull'operatore di negazione.


E' possibile indicare valori numerici, stringa, booleani nel seguente modo: "sringa", [stringa],  32.4 , 32, true.
E' possibile racchiudere espressioni tra parentesi tonde per modificare la priorita' degli operatori.

cicli
I cicli disponibili sono gli stessi del linguaggio C standard:

Ciclo for
for (a ; b ; c) { d };
 cioe': A; finche B : D, C
Funziona a questo modo:
All'inizio del ciclo viene eseguita a: solitamente questa e' la condizione di apertura del ciclo, cioe' quella che setta il valore iniziale della variabile di conteggio.
Prima dell' esecuzione del codice tra parentesi graffe viene valutata l' espressione b: se questa e' falsa il ciclo termina, altrimenti viene eseguito quello che c'e' tra le parentesi graffe.
Una volta eseguito il codice tra le parentesi graffe viene eseguita l'istruzione c, che solitamente viene usata per incrementare il contatore.

Ad esempio:
for (a=4; a<=40; a=a+4)
{ print a."\n";
};
Scrive la tabellina del 4 da 4 a 40. 
La linea a=a+4 assegna ad a il valore contenuto in precedenza su a stesso piu' quattro.

Ciclo while
while (A) {B};

Esegue il codice b finche e' vera la condizione a.
Ad esempio:
a=4;
while (a<=40)
{ print a."\n";
  a=a+4;
 };
E' codice equivalente al precedente, che scrive la tabellina del 4.

Ciclo do-while

do {A} while (B); 

E' analogo al ciclo while, ma la condizone viene valutata dopo aver svolto per la prima volta il ciclo.
Ancora scriviamo il codice della tabellina del 4:

a=4;
do
{ print a."\n"; 
  a=a+4;
} while (a<=40);

I cicli sono tutti equivalenti, e ciascuno puo' essere sostituito con tutti gli altri, ma e'comunque utile avere una certa varieta'.

Condizioni

Le condizioni sono simili a quelle del linguaggio C. 

if (A) B ; 

Se e' vera la condizione A, viene eseguito il codice B.
Ad esempio:
if (a>10) { print "oltre il limite\n"; };

oppure:
if (A) { B }
else (C); 

Se e' vera la condizione A viene eseguito il codice B, altrimenti viene eseguito il codice C.
if (a>10) { print "oltre il limite massimo\n"; }
else { print "nei limiti. Dato accettato.\n"; };
Da notare che prima dell'else non c'e' il punto e virgola, che porrebbe termine alla condizione.

oppure:
if (A) {B}
elseif (C) {D}
else {E};

Se e' vera la condizione A viene eseguito il codice B. Altrimenti viene valutata la condizione C e se e' vera viene eseguito il codice D. Se nessuna delle condizioni A o C e' vera viene eseguito il codice E.

La condizione di un if e' positiva se il risultato dell' if e' un numero positivo.
I cicli vengono trasformati dal compilatore per utilizzare istruzioni di salto. Il risultato e' un programma in cui appaiono solo goto e label: questo serve perche' poi il programma dovra' essere spezzato dalle istruzioni di ask, e quindi e' necessario che queste ritornando non necessitino di uno stack.
Il file compilato dal compilatore ewb e' leggibile, ed e' un applicativo in linguaggio perl. Questo per rendere questo eseguibile indipendente dall' implementazione, e fare in modo che giri su qualunque piattaforma che possa supportare il linguaggio perl stesso. Sono adeguati un windows, un mac, una macchina dos, e un qualunque sistema unix.
Il compilatore ewb e' anch'esso scritto in perl, e quindi puo'essere eseguito su qualunque macchina che supporti perl.  

funzioni e dichiarazione e' possibile dichiarare una nuova funzione nel seguente modo:
sub nomefunzione(variabile ... ,variabile)
{ codice
}
La funzione torna come risultato l' ultimo risultato di elaborazione.
E' allora implementata la FUNZIONE PREDEFINITA return che non fa nulla: semplicemente torna quello che elabora.

La funzione viene chiamata semplicemente con il suo nome seguito dall' elenco dei valori delle variabili.
Ad esempio:
sub prodotto(a,b)
{ var c;
  c=a*b;
  return c;
 };  

Le variabili a e b sono dichiarate implicitamente con la dichiarazione della funzione prodotto.

remove (tabella) cancella ogni elemento il cui primo campo corrisponda al corrente dalla tabella.

var=loadtab (tabella) presente dalla versione 4.01
carica l' intera tabella e la inserisce nella variabile, compresi cancelletti e "\n"
Se si utilizza il motore di database sui files, una tabella di easyweb non e' altro che un file di testo di piu' righe separate da "\n".
ciascuna riga contiene un elenco di valori separati dal carattere cancelletto "#".
Se il testo contiene caratteri cancelletto o caporiga, questi vengono sostituiti con un altro separatore.

exist (tabella) Torna vero se l' elemento il cui primo campo corrisponda esiste nella tabella o torna false.

nome="Enrico";

in (tabella [,condizione])
{};

oppure:
inall (tabella [,condizione])
{}; 

Sono funzioni che servono a svolgere una ricerca all' interno della tabella, e permettono di operare su righe che rispettano una condizione di ricerca.
Ad esempio:
table studenti=matricola, nome, cognome, esame, voto;
inall (studenti, voto>=18)
{ print nome." ha superato l' esame di ".esame." con voto ".voto."\n";
};
'questo primo ciclo torna i risultati di tutti gli esami con esito positivo per tutti gli studenti

in (studenti, voto >=18)
{ print [L' ultimo esame superato da ].nome.[ e' ].esame.[ con voto ].voto.[\n];
};
'questo torna solo le informazioni sull' ultimo esame.
'notate che il codice con le quadre e' piu' leggibile di quello con le virgolette.

Quello che succede nella in e non succede nella inall, e' che il primo elemento viene considerato come chiave della tabella, e quindi ,nel caso esistano piu' elementi con la stessa chiave, cioe' piu' ionformazioni riguardanti uno studente con una stessa matricola, solo l' ultima di esse viene ritornata, e le precedenti vengono considerate come non valide. 

Volendo utilizzare come chiave primaria il primo campo della tabella ma avendo una chiave composta da piu' campi distinti (es: esame+matricola), si ricorda che si puo' inserire in una variabile, e quindi in un qualunque campo della tabella, una lista contenente entrambi i campi senza problemi.

Transazioni
Un utente puo' bloccare piu' volte la stessa tabella. La seconda volta che il blocco verra' tentato, se la tabella e' gia' bloccata dallo stesso utente, questo sara' ignorato, mentre la prima unlock la sblocca.
Ogni volta che sulla tabella va effettuata una add, una remove, o qualunque altra operazione, la tabella viene prima di tutto bloccata.
Questo significa che un utente puo' bloccare piu' tabelle, svolgere su queste una serie di operazioni e poi sbloccarle. E' allora possibile svolgere operazioni sulla base di dati in maniera transazionale, liberandone la parte interessata solo al momento del completamento.
Per informazioni sulle transazioni, vedi "concetti base del linguaggio". 

- funzioni di gestione interfaccia al web -

ask variabile[:tipocampo] [, variabile [:tipocampo]] Mostra a video un form contenente i nomi delle variabili segiti da campi di tipo tipocampo in cui e' inserito il valore corrente della variabile e puo' essere inserito un nuovo valore.
I tipocampo validi sono text, password, "submit", "textarea", "links", "option", "checkbox", "radio", "hidden",  dalla versione 4.0 "file", e dalla versione 5.16 "checks".
Vengono automaticamente inseriti come nascosti i campi contenenti tutte le variabili del programma al momento e la posizione di PC, in modo da poter riprendere da questo punto l' esecuzione al ritorno dalla ask.

Alcuni campi di ask, come per esempio option, radio, submit, possono avere diversi valori.
I valori possono essere inseriti tutti nella stessa variabile separati da punto e virgola.
I campi di tipo option e radio permettono di mettere un asterisco(*) davanti ad uno dei valori per indicare quello selezionato di default.
a="uno;due;tre"; ask a:option;

a="uno;*due;tre"; ask a:option; '(due selezionato di default)


La variabile torna valorizzata ad uno dei tre valori.

Dalla versione 4.00 la ask puo' essere utilizzata anche per inserire dalla macchina cliente un file, in modo da fare un upload sul server.

var f;
ask f:file;

Al ritorno dalla ask, la variabile f contiene il file inserito dall' utente.
Una volta svolti alcuni controlli su questo file (il tipo, la dimensione, etc...), questo puo' essere salvato sul server.
Questo sistema si dimostra utile, ad esempio, per avere un sistema "user friendly" di update delle pagine su un sito web.

Dalla versione 5.16 si puo'chiedere una serie di checkbox a scelta multipla con ask:
a="*uno;*due;tre;quattro";
ask a:checks;
uno
due
tre
quattro
uno e due, che hanno un'asterisco davanti, sono selezionati di default.

Un caso particolare e' la ask per links. Ad esempio:
a="uno;http://www.uniurb.it;tre";
ask a:links;
Costruisce tre links a video, il primo dei quali salta al file "uno", il secondo al sito dell' universita' di Urbino, e il terzo al file "tre".
Per i links puo' essere indicato il nome (cio' che viene visualizzato) e il valore (dove saltare) a questo modo:
a="uno:sintassi.txt;due:http://www.uniurb.it";
ask a;
in questo caso viene visualizzato uno e due, ma si salta rispettivamente a sintassi.txt e a http://www.uniurb.it

Dalla versione 1.3 la ask permette anche di indicare esplicitamente la label che deve apparire a video assieme al campo, anziche' utilizzare il nome del campo stesso.
La sintassi e': 

b=[ok]; ask a:text("Nome e cognome ? "), b:submit;
Tra parentesi, dopo aver specificato il tipo del campo, puo' essere indicato un testo, che puo' contenere anche tags, e che appare a video prima del campo stesso.

showform(variable) svolge una ask con il form indicato nella variabile.
form e showform sono utili per modificare l'impaginazione standard di un programma easyweb. Dovrebbe esserci qualcosa a riguardo tra gli esempi.

show exp   mostra l' espressione indicata come parametro sul web, incorniciata con il template richiesto. Puo' essere visto come l' estensione di una print. 

Esempio: 

show "Hello world by <b>Easyweb</b>";
Se lanciato da un browser genera una pagina html di hello world, con la parola easyweb in grassetto.

Esiste una variabile predefinita format contenente tre valori separati da duepunti.
Il primo di questi valori e' la larghezza di un campo di testo in caratteri, il secondo l' altezza (per campi di testo multiriga, come ad esempio le textarea), e il terzo puo' contenere h o v.
Se contiene h , le varie opzioni dei campi multivalori sono ordinate in orizzontale, mentre se contiene v le stesse opzioni sono ordinate in verticale, l' una sotto l' altra.
Il default e' format="12:5:h";

Format puo' essere utilizzato per modificare l' impaginazione standard di una pagina easyweb, modificando le dimensioni e le posizioni l'uno rispetto all' altro dei campi di input.

Ad esempio:
b="ok;no;forse";
ask b:submit;
b="ok;no;forse";
format="12:5:v";
ask b:submit;

- gestione di liste -
Easyweb fornisce un insieme di funzioni che permettono di utilizzare una variabile come una lista o come un array.

push variabile, valore
Considera la variabile come uno stack, e vi inserisce un elemento, cioe' il valore, che puo' essere ad esempio il risultato di un espressione.
La variabile deve essere vuota prima della prima push [], altrimenti conterra' una valore spurio.

Esempio:
var a;
push a, "uno";
push a, "due";
push a, "tre";  
print a;

Scrive  "756e6f 647565 747265" , che e' la codifica della lista ("uno", "due", "tre").

pop variabile
Recupera dalla variabile, considerata come stack, l' ultimo dei valori inseriti (politica lifo, la testa della lista).
Il seguente codice stampa tutti gli elementi di una lista.

while (a ne "")
{ print pop(a) . "\n";
};

Se applicato alla lista di prima (istruzione push), il risultato e'

tre
due
uno

Cioe' gli elementi vengono estratti dalla lista in ordine inverso all' ordine di inserimento.


numel variabile
Torna il numero di elementi presenti nella variabile, vista come lista.

Ad esempio:
var a;
push a, "uno";
push a, "due";
push a, "tre";
print numel(a)."\n";

Scrive 3, che e' il numero di elementi contenuti nella lista a.

setelem variabile, numero, valore
Cambia il valore dell' elemento di variabile, vista questa volta come un array.
Se viene cambiato il valore di un elemento che passa il limite della dimensione della lista (o dell' array), questa viene estesa per contenerlo.
il numero degli elementi parte da 0.
Cosi' una lista in cui io ho aggiunto l' elemento n°9, conterra' almeno dieci elementi, di cui alcuni possono essere la stringa nulla.

var a;
push a, "uno";
push a, "due";
push a, "tre";
setelem a, 4, "cinque";
print numel(a)."\n";

Stavolta la lista contiene cinque elementi, il cui quarto (posizione 3) e' nullo: ("uno", "due", "tre", "", "cinque"),

getelem variabile numero
Torna l' elemento alla posizione numero della variabile vista come array.
 - Una variabile contenente una lista puo' essere inserita in una lista e cosi' via per ottenere strutture dati di una qualsiasi dimensione.
 
Esempio:
var a;
push a, "uno";
push b, "due";
var b; 
push b, a; 
push b, "tre";
'b e' una lista che contiene la lista a e l' elemento "tre"
'a e' una lista che coniene "uno", "due"
'  ( (uno, due), tre)
print getelem(b, 1)."\n";                       ' scrive "tre"
print getelem ( getelem(b, 0), 1)."\n";  ' scrive "due"

- funzioni per accesso al motore inferenziale -

EasyWeb implementa un motore inferenziale, funzionante dalla versione 0.8.
Questo funziona come nel linguaggio prolog classico, aparte il fatto che le regole possono essere raccolte in gruppi, il cui nome e' una stringa, ad es. "general".

assert ( nomegruppo, regola )   asserisce una regola nel gruppo
Un gruppo di regole e' in realta' una tabella, quindi una volta che una regola e' stata asserita in un gruppo, questa e' visibile a tutte le applicazioni di easyweb che sfruttino quel gruppo di regole la vedranno. 

retract ( nomegruppo, head_regola )  ritratta la regola, cioe' la cancella dal gruppo indicato. Non e' necessario indicare tutta la regola, ma solo il suo funtore. Studiatevi un manuale di prolog.  

goal (gruppo, richiesta)
{ [codice]
}
Svolge il ciclo per tutti i possibili valori di richiesta che rendono vero il goal. Se il ciclo viene svolto almeno una volta torna true, altrimenti torna false.

Un esempio:proviamo ad indicare alcune regole, e vediamo come si possono utilizzare per svolgere delle operazioni.
Ad esempio indichiamo che mario e' piu' alto di gino, che luca e' piu' alto di mario, e che andrea e' piu' alto di gino

assert( altezza, "piualto(mario,gino).");
assert( altezza, "piualto(luca, mario).");
assert( altezza, "piualto(andrea,gino).");

E spieghiamo all' interprete prolog che se A e' piu' alto di B e B e' piu' alto di C, allora A e' piu' alto di C:

assert(altezza,
"piualto(A,C):=piualto(A,B),piualto(B,C).");

Ora svolgiamo un calcolo: chiediamo al compilatore chi e' piu' alto di gino:
goal(altezza, "piualto(gino,X).")
{ print X."\n";
};
Se lanciamo il programma il risultato sara':
luca
mario
andrea

Quindi dalla informazione che mario e' piu' alto di gino e luca e' piu' alto di mario, il programma e' riuscito a decidere che luca e' anche piu' alto di gino.
Questo meccanismo puo' essere utilizzato per sviluppare database che siano in grado di rispondere a questioni semplici (chi e' piu' alto di gino?) , di capire regole e di svolgere, utilizzando queste regole, dei ragionamenti per rispondere anche a domande non direttamente pensata dall' autore del programma.

- funzioni per l' uso delle socket tcp -

Easyweb non e' pensato come un linguaggio per applicazioni inter-processo, ma permette di comunicare con altri programmi tramite socket TCP, o di costruire in maniera semplice un server.

Per creare un processo client, che comunichi con un server, vanno usate le seguenti funzioni:
sock = socket ( hostname, porta )
hostname e' un indirizzo ip o nome di una macchina, porta e' la porta tcp a cui collegarsi.
sock contiene il numero della socket, fornito dal sistema.
a = sread (sock) ;
a contiene una riga letta dalla socket.
swrite (sock, exp);
scrive sulla socket il risultato dell' espressione exp
close (sock);
chiude la socket.

Per creare un processo server il meccanismo e' analogo, con la differenza che va creata una socket di tipo server,
e si rimane su questa in attesa di una connessione.
srvsock = server (porta);
Trasforma il programma in un server in attesa sulla porta.
sock  = accept (srvsock);
attende che un cliente si colleghi. Quando succede sock viene valorizzato con la socket di comunicazione con il client.
La comunicazione con il client, usando sock, e' analoga a quanto visto in precedenza.

Ad esempio scriviamo un programma che stia in attesa di un messaggio su una socket, sulla porta 2000 della nostra macchina, che e' ad esempio prova.w3n.it, con indirizzo 192.168.2.3

Programma "server": in attesa di fornire un servizio sulla porta 2000.

var srvsock, sock, a;
srvsock=server(2000);

while (1)
{ 'ciclo ripetuto per sempre.
   sock=accept(srvsock);
   a=sread(sock);
   print "Ho ricevuto la stringa [".a."]\n";
   close(sock);
};

E scriviamo un programma "cliente" che invii un messaggio sulla macchina prova.w3n.it, alla stessa porta:

var sock;
sock=socket("prova.w3n.it", 2000);
swrite(sock, "C'e' nessiunooo ? \n");
close(sock);

I due programmi sono in grado di comunicare, anche se il programma cliente gira su una macchina diversa dal programma server.

Il meccanismo one-action web

In pratica e' il meccanismo con cui una sola pagina ewb e' in realta' un server che risponde a parecchie richieste http, anche contemporane, e per ciascun utente mantiene una sessione dimodoche' le richieste seguano un ordine logico analogo a quello di un programma con un singolo flusso di esecuzione.
E' comunque da ricordare che il risultato non e' in realta' un programma a singolo flusso di esecuzione, anche quando viene usato da un singolo utente. Il linguaggio quindi prevede meccanismi impliciti per gestire la concorrenza di tabelle, e qualcosa di analogo va fatto per la gestione ad esempio di files.
Il meccanismo viene implementato traducendo un programma ewb in un codice perl piuttosto complesso, con piu' punti di entrata, ciascuno in corrispondenza di una ask, ed un' area iniziale di recupero variabili e salto.

ad ogni lancio della cgi in perl, si controlla una variabile del form: Questa variabile contiene come informazione la posizione a cui saltare per riprendere l' esecuzione.
Lo stesso form contiene anche come nascoste tutte le variabili gia' esistenti al momento della ask.
Questo significa che tutte le variabili dell' esecuzione corrente vengono ripristinate al ritorno dalla ask, e si puo' riprendere dalla riga successiva, indipendentemente da quanto tempo sia trascorso dall' operazione precedente, mentre in linguaggi come asp la sessione scade dopo un tempo limitato. 
I cicli vengono tutti sostituiti con istruzioni di salto condizionato per non avere problemi con lo stack al ritorno dalla ask. Il codice compilato e' costruito con una complessa struttura a stack in modo da poter annidare ask e cicli. 


Gestione dell' interfaccia al web.

la gestione di interfaccia e' sicuramente piu' limitata che con i programmi sviluppati in perl e libcgi.
E' comunque possibile spedire i dati al cliente dopo averli "impacchettati" con un template grafico utilizzando l' istruzione template. E' possibile utilizzare come template sia una stringa indicata esplicitamente sia un file caricato dall' esterno.
Il template deve contenere un tag nullo "< >" che viene tagliato e sostituito dal codice del form o dei forms, e va semplicemente inserito nella variabile predefinita "template".
Il template puo' essere modificato, come codice html,  se viene costruito con l' istruzione form.
Ancora, la grafica dell' applicazione easyweb puo' essere modificata agendo sulla variabile predefinita format, che cambia la forma e la distribuzione dei campi di richiesta a video.

Il preprocessore di EasyWEB

Easyweb implementa un preprocessore analogo a quello del linguaggio C, anche se ridotto.
Le direttive permesse per un programma ewb sono #include, #define, #ifdef, #ifndef

#include permette di inserire all' interno di un programma EasyWEB delle funzioni prese da un altro file.
Puo' essere usato in due modi: con un file che si trova nella stessa cartella del programma o in un percorso specificato, o con un file che si trova nella directory delle librerie di EasyWeb.
La directory predefinita per le librerie di EasyWEB e' "/lib/ewb".
#include < libreria.ewb > cerca la libreria nella cartella /lib/ewb/.
#include " libreria.ewb " cerca la libreria nella directory corrente.

#define permette di definire delle variabili di preprocessore, e di dargli un valore con cui possano essere sostituite all' interno del codice.

#define value 15
print value * 2; ' scrive trenta.

La define puo' essere utilizzata per costruire del codice condizionale, dipendente ad esempio dal sistema operativo su cui il programma dovra' girare. 

#define windows
..
#ifdef windows
codice per wincozz

#else
codice per gli altri sistemi

#endif
