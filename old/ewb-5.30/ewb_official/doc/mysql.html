Supporto per MYSQLe DBI-sqlMYSQL and DBI-sql support  
E' finalmente completa e testata la libreria per il supporto di database su un server mysql, anche su un altra macchina. Finally I completed the library for mysql databases connections, also for a remote database server.
(a dire il vero e' presente da un paio di versioni, ma non avevo trovato il tempo di documentarla.. scusate) The lib is ready from some time, but i never found time to wrote on.

Installazione: Installation:

  • Installate sulla vostra macchina mysql, come lo potete ottenere da http://www.mysql.org . Assicuratevi di aver lanciato il demone, e controllate che l' utility "mysql", cioe' il client testuale, giri. Se usate un server su un altra macchina, vi basta il client testuale. You have to install mysql on tour system. You can obtain it from http://www.mysql.org. You need to launch the daemon, and you need to check wether the utility "mysql" (mysql text-based client) runs. If you use a remote server, you need only the text-based client working.
  • Se siete sotto linux, dovete permettere all' utente di apache di accedere al database: lanciate mysql e scrivete:  If you work on Linux, you need to grant some privilege on database for the apache user. Launch mysql and wrote:
Mysql> create database easyweb_db;
Mysql> grant all on easyweb_db.* to apache@localhost;

potreste dover sostituire localhost con il vostro nome host, o apache con l' utente predefinito del vostro server web. This may change: It could be necessary to wrote your hostname instead of localhost, or "nobody" instead of "apache".
  • sotto windows non so come funzionino gli utenti di mysql, comunque e' sempre meglio creare il database, e e' necessario dare  privilegi di accesso al web server. Under windows, i don't know how mysql users works. You need to create "easyweb_db" database and give access privileges to web users.

Uso:Use:

  •  installate la versione di easyweb 4.14 o successive. Install easyweb version 4.14 or diamond (easyweb 5).
  • compilate aggiungendo l' opzione di compilazione Compile with option -dbmysql
  • esempioExample: ewb -dbmysql prova.ewb 
  • spostate il programma su una cartella del web e usatelo normalmente. Buona fortuna. Move your program on a web folder and use it normally. Good luck.


Limitazioni: le tabelle, che sul database su files hanno i campi di dimensione illimitata, in questo caso limitano i campi a 256 caratteri, oltre i quali i dati vengono troncati. Sorry. Limits: Tables on file-based database engine has fields of unlimited size, in sql the limit of easyweb is 256 chars. Over that limit the fields are cutted. Sorry.

Dalla versione 5.0 sono previste le seguenti modifiche: From version 5.0 there are the following changes:
  • possibilità di usare mysql per collegarsi ad un server di database remoto, anche con utente e password. You can use mysql to connect to a remote database server, also with user and password.
  • possibiltà di usare un qualunque database, e non per forza easyweb_db, che comunque resterà il default.  You can use any database, and not only easyweb_db (the default database for easyweb applications).
  • possibiltà di accedere a database informix o oracle, ed in generale tutto quanto compatibile con DBI di perl. Ad oggi non lo uso in quanto richiede l' installazione di molte librerie, e non la macchina "pulita" come e' filosofia di easyweb. You can use an informix or oracle database, and all databases compatible with perl-DBI. I don't like that because it needs many libraries, and non the only perl interpreter. (work in progress).
